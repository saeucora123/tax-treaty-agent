# Tax Treaty Agent 老师评阅包

日期：2026-03-12  
阶段：`Stage 3.5`  
用途：给专业老师快速看 4 个真实案例输出，判断这个早期原型在专业方向上是否成立，以及下一步更该补什么。

## 1. 先说清楚它是什么

`Tax Treaty Agent` 现在是一个非常早期、范围刻意收窄的国际税协定预审工具原型。

当前只支持：

- 中国 / 荷兰
- 股息、利息、特许权使用费
- 单轮输入

它的目标不是直接替代专业判断，而是做第一轮预审：

- 识别是否在当前支持范围内
- 定位可能适用的协定条款
- 给出可支持的税率上限或结果范围
- 在事实不足时提示缺失事实和下一步动作

## 2. 希望老师重点看的不是完整度，而是这 4 件事

1. 这种“保守的第一轮预审工具”在专业上有没有意义
2. 现在这种输出，是否真的有助于缩小分析范围
3. 如果下一步继续做，更应该优先补“补事实能力”，还是先做更多国家对扩展
4. 哪些地方最容易误导，或者最没价值

## 3. 真实案例

下面案例均来自本地真实接口输出，不是手写示意。

### 案例 A：支持场景 / 股息

输入：

```text
中国公司向荷兰公司支付股息
```

系统识别：

- 付款方国家：`CN`
- 收款方国家：`NL`
- 收入类型：`dividends`

系统输出重点：

- 状态：`预审完成`
- 条款：`Article 10 - Dividends`
- 税率上限：`10%`
- 结论口径：已完成第一轮预审，但仍建议标准人工复核
- 下一步动作：确认条款适用条件、受益所有人事实、持股相关事实

老师可观察点：

- 这个结果是否专业上“有用但不过界”
- 对股息场景来说，当前提示是否足够形成第一轮缩减

### 案例 B：支持场景 / 利息

输入：

```text
中国企业向荷兰银行支付贷款利息
```

系统识别：

- 付款方国家：`CN`
- 收款方国家：`NL`
- 收入类型：`interest`

系统输出重点：

- 状态：`预审完成`
- 条款：`Article 11 - Interest`
- 税率上限：`10%`
- 结论口径：仍属于第一轮预审，不等于最终结论
- 下一步动作：确认利息法律定性、受益所有人、贷款文件和支付记录

老师可观察点：

- 利息场景下，这种输出是否足以支持“先缩小范围，再人工复核”

### 案例 C：信息不足但仍在范围内

输入：

```text
向荷兰公司支付股息
```

系统识别：

- 付款方国家：缺失
- 收款方国家：`NL`
- 收入类型：`dividends`

系统输出重点：

- 状态：`可补全`
- 原因：缺少付款方国家，无法确认交易方向
- 没有强行给出协定结论
- 下一步动作：补充付款方国家或付款方主体信息后重新提交

老师可观察点：

- 这种“保守停下 + 指出缺什么”的处理是否合理
- 这种输出是否真的比简单报错更有价值

### 案例 D：超范围场景

输入：

```text
中国居民企业向美国支付特许权使用费
```

系统识别：

- 付款方国家：`CN`
- 收款方国家：`US`
- 收入类型：`royalties`

系统输出重点：

- 状态：`不在支持范围`
- 原因：当前 MVP 只支持中国-荷兰
- 没有输出实质性协定结论
- 下一步动作：改写成中国-荷兰且属于支持收入类型的查询后再重试

老师可观察点：

- 这种边界拒绝是否清楚且专业
- 是否需要更强的“为什么不支持”说明

## 4. 当前产品已经做到的事

- 已经不是自由聊天机器人，而是结构化预审工具
- 能区分支持、可补全、超范围等不同状态
- 支持结果会给出条款、税率上限、下一步复核动作
- 不完整或超范围时会保守停下，而不是乱猜

## 5. 当前产品还没做到的事

- 还不支持多国家对
- 还不支持多轮补事实
- 还没有完成真实用户反馈后的下一步决策
- 还不是成熟产品，只是专业原型

## 6. 希望老师直接反馈这几个问题

1. 这种工具在专业上有没有实际意义？
2. 现在这种输出结构，是否真的能帮助缩小分析范围？
3. 下一步更该优先补“补事实/追问能力”，还是先扩更多协定覆盖？
4. 哪些地方最容易误导，或者最没有价值？

## 7. 一句话总结

这不是一个“已经完成”的税务产品，而是一个在高风险专业领域里尝试把 AI 放在正确位置的预审工具原型。

这份评阅包想请老师判断的，不是它够不够完整，而是：

`这个方向是否成立，以及下一步最值得补的是什么。`
