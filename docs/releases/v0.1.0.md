# Tax Treaty Agent v0.1.0

![Tax Treaty Agent guided demo](https://raw.githubusercontent.com/saeucora123/tax-treaty-agent/master/assets/tax-treaty-agent-guided-demo.gif)

## What It Is / 这是什么

### 中文

一个面向跨境支付场景的税收协定适用性预审工具。当前聚焦中荷、中新协定下的股息、利息和特许权使用费，以结构化协定数据、保守规则逻辑和 workflow-ready handoff 作为可信度基础。

### English

A pre-screening tool for cross-border payment scenarios under the China–Netherlands and China–Singapore tax treaties. It uses structured treaty data, conservative rule logic, and workflow-ready handoff output rather than runtime model recall.

## What Works Today / 当前可运行范围

### 中文

- 协定范围：China–Netherlands, China–Singapore
- 收入类型：Dividends, Interest, Royalties
- 主路径：wizard-first guided input
- 辅路径：legacy free-text demo lane
- 输出：structured result, treaty citation, BO precheck, review/conflict signals, machine-readable handoff

### English

- Treaty pairs: China–Netherlands, China–Singapore
- Income types: Dividends, Interest, Royalties
- Primary path: wizard-first guided input
- Secondary path: legacy free-text demo lane
- Output: structured result, treaty citation, BO precheck, review/conflict signals, and machine-readable handoff

## How To Verify / 如何验证

### 中文

1. 启动 backend：

   ```powershell
   .\.venv\Scripts\python -m uvicorn app.main:app --host 127.0.0.1 --port 8000 --app-dir backend
   ```

2. 启动 frontend：

   ```powershell
   cd frontend
   npm install
   npm run dev -- --host 127.0.0.1 --port 4173
   ```

3. 运行：

   ```powershell
   python scripts/run_public_smoke.py
   ```

4. 打开 `http://127.0.0.1:4173`，默认 guided workspace 会展示 `CN -> NL dividends` 的原始事实输入面板。

1. Start the backend:

4. Open `http://127.0.0.1:4173`; the app defaults to the guided workspace and exposes the raw-fact `CN -> NL dividends` lane.

## Known Limits / 当前边界

### 中文

- 这不是最终税务意见。
- `MLI/PPT` 当前仅作为 review signal 暴露，不做 runtime 实体判断。
- 不支持当前试点范围外的国家对或收入类型。
- free-text 保留用于 demo / parser validation，不是主产品路径。

### English

- This is not a final tax opinion.
- `MLI/PPT` is exposed as a review signal only, not a substantive runtime override engine.
- Unsupported treaty pairs and income types are rejected explicitly.
- Free-text remains a demo/parser-validation lane, not the primary product path.
